AF-Twin Data Analyst Beta-Version: Mars

This update contains improvements and bug fixes observed during Beta release. It also introduces container technology in order to perform automatic updates without the need for re-installing the application.

Important

  • This update is backward compatible. Databases created with minimum Data Analyst v2.8 will be automatically updated to be compatible with this version.

  • Note that, due to application name change and the first time introduction of the container technology, user are required to uninstall the previous version of the application (i.e. Equinox) and reinstall Data Analyst container.

New Features

  • Application Name Change
    Client application is renamed from Equinox to AF-Twin Data Analyst (shortly Data Analyst).

  • Container Technology
    Application is now embedded inside an App Container. App container is a platform specific implementation and it is responsible for performing automatic updates. It also supplies the Java Runtime to execute the embedded application (i.e. Data Analyst).

  • Server-side Update
    A new database connection pooling engine (DBPool) is added to Equinox Hub Server. With this, the server can be run using one of the 2 possible engines: Hikari or DBPool. This update is intended to solve connection loss issues experienced with the Hikari engine.

  • Encrypted Application Settings
    With this version of Data Analyst, application settings are written to and read from disk using 64bit data encryption.

  • Handling Unconventional FLS Files
    Reading tolerance is introduced in order to handle unconventionally formatted FLS files. It is observed that some of the FLS files did not have space separator between typical flight name and severities. This prevented Data Analyst to read the files. With the introduction of tolerance reader, application reads the files but warns the user for unconventional formatting.

  • Uploading Container & Application Updates
    Administrators can now upload container and application update packages to server via the Data Analyst Admin menus.

Corrections & Bug Fixes

  • Bug Fix: Bug Reporting
    Bug reporting system was broken in the previous version. The reason for the bug is the buffer-overflow during network communication due to excessive bug report sizes. Problem is resolved by splitting bug reports and sending the partitions over the network.

  • Bug Fix: Equivalent Stress Analysis with Sigma File Input
    Equivalent stress analysis did not work when Sigma file was supplied as input. Problem was due to wrong table reference to the local database. Bug is fixed.

  • Bug Fix: User Interface Theme
    Application did not start if user changed the UI theme from settings menu. This was due to themes other than the default theme not being included in the application package. Problem resolved.

  • Search Pilot Points
    A feedback that the search results are limited to the top 100 and the order of results is given at the top of results panel.

  • Airbus Logo
    Airbus logo blocked the last item in the files list in the previous versions. Issue is resolved.

New Plugins

No new plugin is added in this release.
